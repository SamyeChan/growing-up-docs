## JavaScript 知多少
### 咩系 JavaScript 啊？

- 世界上最流行的 **脚本语言**；
- 一种运行在浏览器中的 **解释型** 编程语言；

### 做咩要学 JavaScript 啧？

简单来说，因为你没得选...在Web世界里，只有JavaScript能跨平台、跨浏览器驱动网页，与用户交互。作为前端，就酱紫好好地被安排吧。

### JavaScript 仔仔诞生记

1995年的美国，凭借 `Navigator` 浏览器正当红的 **网景公司** 为了在静态 HTML 网页上添加一些有趣的动态效果，就派 `Brendan Eich` 小哥，在2周的时间里将 `JavaScript` 仔仔设计了出来。而为什么叫 Java～～～Script，大概就是如今人们常说的 **蹭...流...量...** 吧。总之，`JavaScript` 仔仔就在10天的时间里匆忙诞生了～

可能效果还不错，一年后，微软又模仿着 `JavaScript` 仔仔设计出了 `JScript`...这样明显不太好，还有没有王法啦？？？于是为了 **统一标准**，几家公司联合着 `ECMA`（European Computer Manufacturers Association）定制了 `JavaScript` 仔仔的标准，也就是后来越来越被人提起的 `ES`（ECMAScript标准）啦。

简单点就是，ECMAScript 是一种语言标准，而 JavaScript 是网景公司对 ECMAScript 标准的一种实现，然后很多情况下，`JavaScript` == `ECMAScript`。

## 关于编写 JavaScript 的注意点

- 不可以用 **Word** 或 **写字板** 来编写 JavaScript 或 HTML（带格式的文本保存后不是纯文本文件，无法被浏览器正常读取）；
- 尽量不要用 **记事本** 编写（它会自作聪明地在保存UTF-8格式文本时添加BOM头）；
- 由于浏览器的安全限制，以 `file://` 开头的地址无法执行如联网等 JavaScript 代码；

## 基本语法

- 语法和 Java 类似，每个语句以 `;` 结束（不强制，JS引擎会自动补全），语句块用 `{...}`；
- JavaScript **严格区分大小写**；
- JavaScript 允许对任意数据类型做比较 → 比较运算符 `==` & `===`

比较运算符 | 描述
- | -
== | 会自动转换数据类型再作比较，不推荐
=== | 不会自动转换数据类型，数据类型不一致就 false，一致再进行下一层比较

## 数据类型和变量

最新的 JavaScript 数据类型如下（9种）：

- 值类型(基本类型)：字符串（String）、数字(Number)、布尔(Boolean)、对空（Null）、未定义（Undefined）、Symbol。

- 引用数据类型：对象(Object)、数组(Array)、函数(Function)

#### Number

- 不区分整数和浮点数，统一使用 Number 表示；
- 可直接做四则运算，规则与数学一致；
- `NaN` 是一个特殊的 Number，它和谁都不相等，包括自己 → 唯一能判断 `NaN` 的方法就是通过 `isNaN()` 函数；
- 浮点数的比较 → 浮点数在运算过程中会产生误差（计算机无法精确表示无限循环小数）→ 要比较两个浮点数是否相等，只能计算它们之差的绝对值，看是否小于某个阈值；



#### 字符串

- 单引号 `''` 或 双引号 `""` 括起部分；
- 转义字符；
- 模版字符串；
- 常见方法（返回新字符串，不改变原有字符串）：

1. `toUpperCase()` - 全大写；
2. `toLowerCase()` - 全小写；
3. `indexOf()` - 返回指定 **字符串索引 + 1**；
4. `substring()` - 返回指定索引区间子串；

```js
str.substring(开始索引值，结束索引值 + 1)

// 不接受负参数
// 若前参数 > 后参数，将交换位置
```

#### 布尔值

- `true` / `false`；
- 经常使用在条件判断中；

#### null & undefined

- `null` 表“空”值，`0` 表数值0，`''` 表长度为0的字符串；
- `undefined` 表示值未定义；
- 大多数情况下，用 `null` 吧，`undefined` 仅在判断函数参数是否传递的情况下有用；

#### 数组

- 一组按序排列的集合，提供一种顺序存储一组元素的功能，并可以按索引来读写
- JavaScript 数组可包括任意数据类型；
- 数组创建：`new Array()` / `[]`（推荐）；
- 直接给 `arr.length` 赋值，会改变 `arr` 大小；
- 通过索引赋值时，若索引超过数组原范围也会改变 `arr` 大小；
- 常用方法：

1. `indexOf()`- 返回一个指定元素的位置；
2. `slice()` - 截取数组部分元素，返回新数组；

```js
arr.slice(开始索引值，结束索引值 + 1)

// substring 的数组版
```

3. `push()` - 向数组末尾添加元素（可多个）；
4. `pop()` - 删除数组最末尾元素（一个）；
5. `unshift()` - 向数组头部添加元素（可多个）；
6. `shift()` - 删除数组最头部元素（一个）；
7. `sort()` - 排序（直接修改元素位置）；
8. `reverse()` - 将数组元素反转；
9. `splice()` - 修改数组的“万能方法”（在指定元素处可删可增，改变原始数组）

```js
arr.splice(执行位置, 删除数量, 新增项目)
```
10. `concat()` - 连接数组（返回新数组）；
11. `join()` - 将数组以指定字符连接为字符串；

**数组也是一个对象**，其每个元素的索引被视为对象的属性；


#### 对象

- 一组由 `键 - 值`组成的无序集合；
- JavaScript 对象的健都是字符串类型，值可任意；
- JavaScript 规定，访问不存在的属性不报错，而是返回 `undefined`；
- JavaScript 对象是动态类型，故可自由地给一个对象添加或删除属性；
- 检测对象是否拥有某一属性，使用 `in` 操作符：

```js
'你想查的属性名' in 对象

// 继承的属性亦可判断，并返回一个 true
```
- 判断对象自身拥有属性：

```js
obj.hasOwnProperty('你想查的属性名')
```

#### 变量

- 在计算机程序中，变量不仅可以是数字，还可以是任意数据类型；
- 大小写英文、数字、$ 和 _ 的组合，数字不可首位；
- 一个变量若未通过 `var` 申明就被使用，那么该变量就自动被声明为全局变量；

#### strict 模式

- 在 `strict` 模式下运行的 JavaScript 代码，强制通过 `var` 声明变量，未使用 `var` 申明变量就使用的，将导致运行错误；
- `strict` 模式的启用，在 JavaScript 代码首行写入：

```js
'use strict';
```
（这是一个字符串，不支持 strict 模式的浏览器会把它当做一个字符串语句执行，支持 strict 模式的浏览器将开启 strict模式运行JavaScript）

## 循环

- `for循环` - 常用于利用索引遍历数组；
- `for...in` - 将对象属性依次循环；

（若想过滤掉对象继承的属性，则使用 `hasOwnProperty()` 做一层判断实现）
（`for ... in` 对 `Array` 的循环得到的是 `String` 而不是 `Number`）

- `while` & `do...while`

循环是让计算机做重复任务的有效的方法，但死循环会让浏览器无法正常显示或执行当前页面的逻辑，甚至直接挂掉。

## Map & Set

- JavaScript 的默认对象表示方式 `{}` 可视为其他语言中的 `Map` 或 `Dictionary` 的数据结构，即 **一组键值对**；
- ES6 新数据类型 - 解决 JavaScript 对象属性必为字符串问题；

#### Map

- 一组键值对的结构，具有极快的查找速度；
- “根据名字快速查找数据”；

#### Set

- 一组无重复key的集合，且不存储value；
- 创建： `new Set()` / `new Set([...])` - 重复元素会被自动过滤；
- 添加元素： `add(key)`；
- 删除元素： `delete(key)`；

## Iterable

- ES6 新类型 - 为统一集合类型而生；
- `Array`、`Map` 和 `Set` 都属于 `iterable` 类型；
- 具有iterable类型的集合可以通过新的 `for ...of`循环来遍历
- for...in 、 for...of 、 forEach

```js
obj.forEach(item/val, index/key, 对象本身) { ... }
```

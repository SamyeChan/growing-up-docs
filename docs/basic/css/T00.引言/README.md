# 【引言】

- 如今的挑战：在保证DRY、可维护性、灵活性、轻量级并且尽可能符合标准的前提下，把我们手中的CSS特效转化为网页中的各种创意；
- 核心目的：如何用CSS解决难题；
- **理解发现解决方案的过程比解决方案本身更有用**；

## 一些概念/备注

- Atlas
- Antenna House
- git用于版本控制；
- SCSS用于CSS预处理；
- Espresso（文本编辑器）；
- CodeKit用于把SCSS编译成CSS；
- Dabblet用于存放在线演示；
- Adobe Illustrator；
- Modernizr；
- `@supports` 规则实现回退（当想尝试的某个CSS特性十分新时）→ 但必须慎用；
- 若要检测某个样式属性是否被支持，核心思路：**在任一元素 的element.style对象上检查该属性是否存在**；
- 浏览器可以解析某个CSS特性，并不代表它已经实现（或正确实现）了这个特性；

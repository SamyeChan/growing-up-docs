## 二清

- 即 **二次清算**；
- 支付机构 → 平台 → 商户/客户；

支付机构将资金结算给平台，平台再自行将资金清算给下游的商户/客户

- 若平台无清算资质（无支付牌照），即违规二清；
- 需防范平台吸收资金后挪用其进行高风险投资，甚至跑路；

## 支付牌照

- 即 **《第三方支付的业务许可证》**，央行颁发，并受其监管；
- 三种牌照：银行卡收单（POS机）、互联网支付、预付卡发行与受理；

![首批获牌企业](./imgs/note-pay.png)

- 支付牌照之所以抢手，是因为其申请门槛极高，需央行审核，并且，央行几乎已停止发牌照了；
- 目前全国仅存200家左右的第三方支付公司，而可做互联网支付的或许不到100家了；
- “备付金”？！

## 如何避免二清？

1. 最好的方法：有支付牌照（难）；
2. 第三方支付公司、有开发能力银行的分账系统；
3. 微信分账系统；

## 代收和代付

#### 代收

- 亦即 **代扣**、**免密支付**；
- 用户签署协议授权企业可以直接从用户账户中扣钱；
- 整个环节无需用户输密码或验证，接口敏感；
- 开放接口难度大（需为深度合作企业）；

#### 代付

- 企业向下属用户或商家付款；
- 出款账户一般是在机构开立的备付金专户；
- 常用于结算、提现等；

## T+1 和 D+1

概念 | 解释
- | -
T+1到账 | 今日的交易金额在下一个工作日到账
D+1到账 | 今日的交易金额明天到账

二者区别：是否区分工作日

## 长款 和 短款

- 完善的支付系统都需要有对账系统的辅助，来及时发现系统的差错，核实资金；
- 对账的最终结果有两种：

1. **长款** - 钱收多了；
2. **短款** - 钱收少了；

## 重复支付 和 无效支付

- 因网络或其他因素导致的，在不同平台上进行同一订单的多次支付（可能会 “多付单货”，也可能 “只付没货”）
- 兜底的方案：发起原路退款，根据实际业务场景决定退款是否需要人工审核；

## 银联 和 网联

#### 共同点

二者都为中国的支付清算组织，由中国人民银行批准成立。

#### 主要差异

- 银联：倾向于管理线下银行卡收单；

中国银联是中国银行卡联合组织，全国两千多家商业银行，银联搭建起了一个可以跨行交易清算的系统；

- 网联：倾向于管理支付宝微信等移动支付；

第三方支付机构由网联作为中间平台转接银行，而非直接对接银行 → 断直连；

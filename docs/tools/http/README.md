# 网络 | HTTP

## HTTP 是什么

- = HyperText Transfer Protocol，超文本传输协议；
- 一种可扩展的网络传输协议 → 应用层协议；
- 基于 TCP/IP 通信协议传递数据；
- 在 Web 上进行数据交换的基础，是 `client-server` 协议；
- 所有 `WWW` 文件都必须遵守这个标准；
- 客户端发出的消息为 `requests`，被服务器响应的消息为 `responses`；

## HTTP 组成 & 特点

- HTTP = 请求报文 + 响应报文；
- HTTP请求 = 请求行 + 请求头部 + 空行 + 请求数据；
- HTTP响应 = 状态行 + 消息报头 + 空行 + 响应正文；
- 特点：简单快速 / 灵活 / 无连接 / 无状态；

## HTTP 请求方式

8种

1. OPTIONS
2. HEAD
3. **GET**
4. **POST**
5. PUT
6. DELETE
7. TRACE
8. CONNECT

## HTTP 协议

- Hyper Text Transfer Protocol，超文本传输协议；
- 位于 TCP/IP 四层模型中的应用层；

![TCP/IP 四层模型](./imgs/0000_http.png)
- HTTP 协议通过请求/响应的方式，在客户端和服务端之间进行通信；

![HTTP 协议的请求/响应方式](./imgs/0001_http.png)
- BUT...HTTP --> 不够安全；
  1. HTTP 协议的信息传输完全以明文方式进行 --> 网络裸奔；
  2. 中间人攻击：信息被恶意拦截甚至篡改；
  3. 对称加密 - 拦截第一次通信就可以知道密钥了哇；
  4. 非对称加密 - 拦截服务端公钥key1，换成自己的key，然后给客户端加密，客户端用错误的key加密自己的key2，返回，又拦截，解密就拿到key2了，这个key2可是日后进行对称加密的重要“钥匙”噢～

## HTTPS 协议

- 服务端把密钥发给证书机构，换一个证书，日后就用证书通信啦～
- 客户端要验证证书 --> 各大浏览器和操作系统已经维护了所有权威证书机构的名称和公钥 --> 客户端只需知道时哪个机构颁布的证书，就可以在本地找到对应的机构公钥，解密出证书签名；
- HTTS 在 HTTP 协议的基础上增加了 SSL 安全层 --> 上述一系列认证就是在 SSL 层中完成的；

![SSL 层](./imgs/0002_http.png)
- 最新推出的 TLS 协议，是 SSL 3.0协议的升级版，和 SSL 协议原理相似；
